STEP 1 — Backend
─────────────────
Open Terminal / Command Prompt and run:

  cd taxai-widget/backend

  # Create virtual environment
  python -m venv venv

  # Activate it
  # Mac/Linux:
  source venv/bin/activate
  # Windows:
  venv\Scripts\activate

  # Install dependencies
  pip install -r requirements.txt

  # Copy env file (leave values blank for no-key mode)
  cp .env.example .env

  # Start the server
  python -m uvicorn main:app --reload --port 8000

You should see:
  ✓ LLM provider : Rule-based NLP (no API key found)
  ✓ Storage      : In-memory (no MONGODB_URI set)
  ✓ TaxAI backend ready

Test it: open http://localhost:8000/health in browser
→ Should show: {"status":"ok"}


STEP 2 — Frontend
──────────────────
Open a SECOND terminal window and run:

  cd taxai-widget/frontend

  # Install Node packages
  npm install

  # Start Angular dev server
  npm start

Open browser at: http://localhost:4200
→ You should see the TaxAI widget


STEP 3 — On Codeathon Day (add real credentials)
──────────────────────────────────────────────────
Open backend/.env and fill in:

  AZURE_OPENAI_KEY=<key from organizers>
  AZURE_OPENAI_ENDPOINT=https://YOUR_RESOURCE.openai.azure.com
  AZURE_OPENAI_DEPLOYMENT=gpt-4.5
  AZURE_OPENAI_API_VERSION=2024-02-01

  MONGODB_URI=<connection string from organizers>
  MONGODB_DB=taxai

Restart backend → log will confirm GPT-4.5 and MongoDB are active.

 HOW TO SWAP THE TAX ENGINE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Only ONE file changes: backend/tax_engine.py
Everything else (Angular, FastAPI, adapters) stays the same.

Current engine: US Sales & Use Tax (state rates, nexus, exemptions)

To switch to Federal Income Tax:
  1. Replace backend/tax_engine.py with the federal version
     (keeps same TaxInput / TaxResult class names)
  2. That's it. No other file changes needed.

To switch to a third-party API (TaxJar / Avalara):
  1. Replace backend/tax_engine.py with a version that
     calls the external API instead of local logic
  2. That's it. No other file changes needed.

The contract the router expects from tax_engine.py:
  - Class:    TaxInput
  - Function: calculate_sales_use_tax(TaxInput) → TaxResult
  - TaxResult must have: tax_amount, rate_pct, state_name,
    tax_type_label, total_due, has_nexus, nexus_reason,
    is_exempt, exemption_reason, notes (list)

If you also change the LLM extraction prompt (for federal tax):
  - File: backend/providers/llm/azure_openai_adapter.py
  - Edit: EXTRACT_PROMPT variable (top of file, ~15 lines)
  - This is optional — only needed if field names change
